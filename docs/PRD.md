# 对话聚类分析工作台 PRD

## 1. 产品概述

### 1.1 产品名称

对话聚类分析工作台

### 1.2 产品定位

这是一个本地运行的单用户 Web 工具，用来把业务对话中的开放式文本分析流程工具化，形成一套可持续演进的任务级类别体系。

它不是一次性聚类脚本，而是一个支持“提取、建类、分类、收敛、导出”的分析工作台。

### 1.3 当前版本目标

- 把开放式文本分析从手工操作沉淀成稳定流程
- 让类别体系可以持续迭代，而不是每次重新开始
- 保留完整过程日志，支持排查和回溯
- 输出结构化结果，支持汇总和导出
- 具备一定泛化能力，不只局限于单一业务语境

### 1.4 目标用户

- 销售运营
- 销售管理者
- 增长分析人员
- 用户研究人员
- 需要从大量业务对话中提炼模式和类别的个人使用者

## 2. 解决的问题

现实中的业务对话分析通常存在这些问题：

- 对话量大，人工读不完
- 类别体系容易漂移
- 每轮分析过程不可追溯
- LLM 对话结果难复用
- 导出结果结构不统一

本产品的目标，是把这件事变成一套本地可运行、可检查、可修正、可导出的工作流。

## 3. 核心对象

### 3.1 任务

一个任务代表一套独立的分析体系。

一个任务下有：

- 多个批次
- 一套任务级类别表
- 当前分析结果
- 执行日志

### 3.2 批次

一个批次对应一次 CSV 上传，是主要的执行单元。

当前批次分两种用途：

- `seed`：建类批次
- `classify_only`：直接分类批次

### 3.3 对话

导入后的标准化原始记录，至少包含：

- 原始 `id`
- 原始 `text`
- 去重哈希

### 3.4 类别表

任务级长期资产，包含：

- 类别名称
- 类别定义
- 系统类别 `其他`
- 任务收敛过程中新增或合并后的类别

### 3.5 当前分析结果

每条对话当前生效的结构化结果，包括：

- 分析摘要
- 当前类别
- 判断依据引用原文
- 判断依据说明

### 3.6 过程日志

保留每一次：

- 信号提取
- 聚类建议
- 批量分类
- 处理全部其他
- 合并近似类别

对应的执行记录和 LLM 调用日志。

## 4. 输入与输出

### 4.1 输入

- UTF-8 编码 CSV
- 默认按前两列读取：
  - 第 1 列作为 `id`
  - 第 2 列作为 `text`
- 支持一次上传最多 10 个 CSV

### 4.2 输出

每条对话最终输出：

- 原始对话 ID
- 原始对话文本
- 分析摘要
- 类别
- 判断依据引用原文
- 判断依据说明
- 当前结果状态

任务级输出：

- 各类别数量
- 各类别占比
- 类别样例
- 当前任务 CSV 导出

## 5. 核心流程

### 5.1 主流程

1. 新建任务
2. 上传一个或多个 CSV 批次
3. 用建类批次做信号提取
4. 基于提取结果生成聚类建议
5. 人工确认建议，写入正式类别表
6. 对后续批次按当前类别表做批量分类
7. 当 `其他` 变多时，执行任务级“处理全部其他”
8. 必要时执行“合并近似类别”
9. 查看汇总结果并导出

### 5.2 处理全部其他

`处理全部其他` 已经确定为任务级动作，而不是按批次分别处理。

原因：

- 避免多个批次各自处理 `其他` 时生成含义相近但名字不同的类别
- 用统一池子聚类，更容易收敛类别体系

处理方式：

1. 汇总当前任务下所有属于 `其他` 的记录
2. 再次提取这些记录的分析信号
3. 基于这些摘要再次聚类
4. 确认新的类别建议
5. 只对这些记录重新分类

### 5.3 合并近似类别

任务收敛区支持近似类别合并：

1. 用户设置最大合并后类别数
2. LLM 生成合并建议
3. 用户确认后应用
4. 历史结果同步迁移到新类别
5. 被合并的旧类别标记为 `inactive`

## 6. 功能需求

### 6.1 首页

- 已有任务列表
- 新建任务
- 模型与并发配置
- LLM Prompt 配置

### 6.2 任务管理

- 创建任务
- 查看任务列表
- 删除任务
- 进入任务工作台

### 6.3 CSV 导入

- 上传 UTF-8 CSV
- 默认按前两列读取 `id/text`
- 支持多文件批量上传
- 任务内去重
- 批次列表展示导入结果

### 6.4 批次推进

- 全行可点击切换批次
- 记住最近一次选中的批次
- 建类批次和直接分类批次有不同操作路径

### 6.5 信号提取

- 对建类批次执行逐条提取
- 支持失败重试
- 提取完成后先展示样例，再进入下一步

### 6.6 聚类建议

- 基于提取结果生成类别建议
- 用户自己决定何时开始聚类
- 聚类后支持：
  - 确认写入类别表
  - 废弃建议并回退

### 6.7 批量分类

- 对批次逐条分类
- 支持重新批量分类
- 无法归类时落入 `其他`

### 6.8 类别表管理

- 在 Category Snapshot 里行内编辑
- 新增类别
- 删除类别
- 系统类别 `其他` 只允许编辑定义，不允许删除

### 6.9 处理全部其他

- 任务级收敛 `其他`
- 显示当前轮次状态
- 支持查看当前轮次提取 / 聚类 / 重分状态

### 6.10 汇总与导出

- 汇总各类别数量和占比
- 饼图展示
- CSV 明细导出

### 6.11 日志与追踪

- 保存 step run
- 保存 step run item
- 保存 LLM 调用日志
- 高亮失败项
- 高亮未完整落库的失败 case

## 7. 页面结构

### 7.1 首页

从上到下：

1. 已有任务
2. 新建任务
3. 模型与并发配置
4. LLM 提示词配置

其中：

- `新建任务`：当没有任务时默认展开，否则默认收起
- `模型与并发配置`：默认收起
- `LLM 提示词配置`：默认收起

### 7.2 任务工作台

- 顶部：任务总览
- Tab 1 `批次推进`
  - 批次表
  - Batch Detail
  - 上传 CSV
- Tab 2 `任务收敛`
  - Category Snapshot
  - 处理全部其他
  - 合并近似类别
  - 汇总展示与导出

## 8. 数据存储

本期使用本地 SQLite 文件持久化。

主要表：

- `tasks`
- `batches`
- `dialogs`
- `categories`
- `dialog_analysis_results`
- `step_runs`
- `step_run_items`
- `llm_call_logs`
- `category_suggestions`
- `category_merge_suggestion_items`
- `category_merge_mappings`
- `app_settings`

## 9. 当前版本状态

### 9.1 已实现

- 首页与任务列表
- 新建任务
- 多文件 CSV 导入
- 批次用途管理
- 信号提取与失败重试
- 聚类建议与建议废弃
- 批量分类与重新分类
- 处理全部其他
- 合并近似类别
- Category Snapshot 行内管理
- 日志页
- 汇总展示与导出

### 9.2 当前限制

- 本地单用户，不支持多人协作
- SQLite 适合个人使用，不适合共享并发写入
- 聚类建议和合并近似类别仍依赖模型质量
- 少量情况下仍可能出现底层调用“半截中断”，虽然已有自动收口机制
