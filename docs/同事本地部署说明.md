# 同事本地部署说明

这份文档是给“几乎没有 coding 基础”的同事准备的。

目标只有一个：

- 把这个工具装到你自己的电脑上
- 成功打开网页
- 能开始使用

如果你不懂下面这些词，没有关系：

- Git
- Node.js
- npm
- 终端 / Terminal / PowerShell

你只需要按顺序照做，不要跳步。

---

## 先说结果：装好以后你会看到什么

当部署成功时，你会在浏览器里打开这个地址：

```text
http://localhost:3000
```

然后看到这个工具的首页。

只要你能看到首页，就说明已经部署成功了。

---

## 你需要准备什么

开始前请先确认，你手上有这 3 样东西：

1. GitHub 仓库地址  
例如：

```text
https://github.com/chairlong-create/Cluster-Analysis
```

2. 模型 API 信息  
至少要知道你要使用哪一家模型，以及对应的：

- API Key
- Base URL
- Model 名称

3. 一台你自己的电脑  
支持：

- mac
- Windows

---

## 整体流程一共 9 步

你可以把它理解成：

1. 安装两个基础软件
2. 把项目下载到电脑里
3. 安装项目依赖
4. 启动项目
5. 在浏览器里打开
6. 先确认网页能跑起来
7. 在网页里配置模型和提示词
8. 正式使用时切到更稳的启动方式
9. 做一次最小验证

---

# A. Mac 用户安装步骤

## 第 1 步：安装 Node.js

### 这一步是干什么的

这个项目必须依赖 Node.js 才能运行。

本项目建议版本：

- `Node.js 24+`
- `npm 11+`

建议补充：

- 如果你只是日常使用这个工具，不改代码，后面尽量用“正式启动方式”
- 不要长期使用开发模式，否则执行任务时电脑可能会明显变卡

### 怎么安装

1. 打开官网：

[https://nodejs.org/](https://nodejs.org/)

2. 下载 `Node.js 24` 或更高版本  
如果页面有多个版本，优先选：

- `LTS`

3. 下载完成后，双击安装包，一路点继续即可

### 怎么确认安装成功

安装完成后，打开 mac 自带的 `终端（Terminal）`

输入：

```bash
node -v
```

再输入：

```bash
npm -v
```

如果都能看到一串版本号，例如：

```bash
v24.x.x
11.x.x
```

就说明这一步成功了。

### 如果你的电脑里还没有安装 Node.js / npm，怎么办

不用单独再安装 npm。  
正常情况下，只要你安装了 Node.js，npm 会一起装好。

如果你输入：

```bash
node -v
npm -v
```

看不到版本号，或者版本太低：

- 重新去 [https://nodejs.org/](https://nodejs.org/) 下载最新版 LTS
- 安装完成后，先关掉终端，再重新打开一次
- 再次执行 `node -v` 和 `npm -v`

如果 `node -v` 有版本号，但 `npm -v` 没有：

- 通常是安装不完整
- 直接重新安装一遍 Node.js，一般就能解决

---

## 第 2 步：安装 GitHub Desktop

### 这一步是干什么的

它可以帮你把 GitHub 上的项目下载到电脑里。  
对小白来说，这比命令行更容易。

### 怎么安装

打开：

[https://desktop.github.com/](https://desktop.github.com/)

下载并安装 `GitHub Desktop`

### 装好以后做什么

1. 打开 `GitHub Desktop`
2. 登录你的 GitHub 账号

---

## 第 3 步：把项目下载到电脑里

### 怎么做

1. 打开 `GitHub Desktop`
2. 点击顶部菜单：

```text
File -> Clone repository
```

3. 在 URL 位置输入仓库地址：

```text
https://github.com/chairlong-create/Cluster-Analysis
```

4. 选择你要保存到电脑的文件夹
5. 点击 `Clone`

### 你会看到什么

下载完成后，你会在电脑里得到一个文件夹，名字一般是：

```text
Cluster-Analysis
```

---

## 第 4 步：打开项目文件夹

在 Finder 里找到刚刚下载好的 `Cluster-Analysis` 文件夹。

后面所有操作，都在这个文件夹里完成。

---

## 第 5 步：安装项目依赖

### 怎么做

1. 打开 `终端（Terminal）`
2. 输入 `cd`，后面加一个空格
3. 把 `Cluster-Analysis` 文件夹直接拖进终端窗口
4. 按回车

你会进入项目目录。

然后输入：

```bash
npm install
```

### 这一步可能要等多久

通常 1 到 5 分钟。

### 怎么算成功

如果最后没有报红色错误，并且终端停下来可以继续输入命令，就算成功。

---

## 第 6 步：先把项目跑起来

如果你只是想先确认能不能打开网页，可以先用下面这个方式启动：

在终端里输入：

```bash
npm run dev
```

如果成功，你通常会看到类似：

```text
http://localhost:3000
```

然后：

1. 打开浏览器
2. 输入：

```text
http://localhost:3000
```

3. 回车

如果能看到首页，说明项目已经成功跑起来了。

但是注意：

- `npm run dev` 是开发模式
- 它适合第一次检查项目能不能启动
- 如果后面你要长期使用，建议改用下面第 8 步的“正式启动方式”

---

## 第 7 步：进入工具页面后，再配置模型和提示词

这个工具已经提供了前端页面，不需要你在终端里手动配置模型和提示词。

你需要做的是：

1. 先让网页成功打开
2. 在工具自己的页面里找到模型配置入口
3. 在页面里填写：
- API Key
- Base URL
- Model 名称
4. 如有需要，再在页面里编辑提示词

### 你只需要知道这 3 个概念

- `API Key`：相当于密码
- `Base URL`：模型接口地址
- `Model`：模型名字

### 常见错误

1. API Key 填错  
会导致模型请求失败

2. Base URL 填错  
会导致请求发不出去

3. Model 名称填错  
会导致模型接口报错或没有返回结果

---

## 第 8 步：正式使用时，建议改成这个启动方式

如果你已经确认网页能打开，后面日常使用建议不要继续用 `npm run dev`。

请改用：

```bash
npm run build
npm run start
```

然后打开：

```text
http://localhost:3000
```

### 为什么更建议这样做

因为：

- `npm run dev` 是开发模式
- `npm run start` 是更适合正式使用的模式
- 如果你在本机执行批量任务，正式模式通常会比开发模式更稳、更不卡

---

## 第 9 步：验证是否真的可以使用

建议你至少做两件事：

1. 能正常打开首页
2. 在页面里完成一次模型配置，并尝试跑一个最小任务

如果首页能打开，但模型请求失败，通常不是“项目没装好”，而是“模型配置还没填对”。

# D. 最常见的 6 个问题

## 1. 输入 `node -v` 报错

说明 Node.js 没装好。  
回到第 1 步，重新安装 Node.js。

## 2. 输入 `npm install` 报错

通常先检查：

- Node.js 版本够不够新
- 你是不是已经进入项目目录

## 3. 运行 `npm run dev` 或 `npm run start` 报错

通常先检查：

- 你是不是已经先执行过 `npm install`
- Node.js / npm 版本是否满足要求

## 4. 页面能打开，但提取失败

先看两件事：

- 你在前端页面里填写的模型配置是不是对的
- 任务日志页里的 `response_text` 写了什么

## 5. 我不知道项目目录在哪

回到 GitHub Desktop，看你 clone 项目时选的保存位置。

## 6. 我没有安装 Node.js / npm

直接回到第 1 步。

你需要做的不是找别人发给你安装包，而是：

- 去 [https://nodejs.org/](https://nodejs.org/) 下载最新版 `LTS`
- 安装完成后重新打开终端或 PowerShell
- 执行 `node -v` 和 `npm -v` 检查版本
